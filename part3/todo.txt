installer docker

installer k3d:
wget -qO - https://raw.githubusercontent.com/k3d-io/k3d/main/install.sh | bash

installer kubetcl :
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl
sudo mv kubectl /usr/local/bin/

Utilise cette commande pour que le port 80 et 8888 de ton ordi communiquent avec le cluster:
k3d cluster create me-cluster --agents 2 -p "8080:80@loadbalancer" -p "8888:8888@loadbalancer"

Installer argoCD:
- Crée un namespace dédié : 
kubectl create namespace argocd
- Applique le script officiel :
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

Par défaut, Argo CD est "caché" dans le cluster. Pour y accéder depuis ton navigateur :

Port-forward : kubectl port-forward svc/argocd-server -n argocd 8443:443

Va sur https://localhost:8443.

Login : admin.

Password : Récupère-le avec cette commande :
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d

La vérification (Le test final)
Une fois qu'Argo CD affiche "Synced" en vert :

Vérifie que tes Pods tournent : kubectl get pods -n dev.

Modifie ton fichier deployment.yaml sur GitHub (passe à replicas: 3).

Attends (ou clique sur "Refresh" dans Argo CD).

Magie : Tu verras les 2 nouveaux Pods apparaître tout seuls dans l'interface.


//////////////////////////////

# 1. Une seule fois — installer les outils
./setup.sh

# 2. Construire tout le cluster
make build
# → crée le cluster k3d
# → crée les namespaces argocd et dev
# → installe ArgoCD dans le cluster
# → attend que ArgoCD soit prêt
# → applique confs/application.yaml (lien vers GitHub)
# → affiche le mot de passe admin

# 3. Dans un AUTRE terminal — accéder à l'UI
make forward
# → ouvre https://localhost:8443 dans ton navigateur
# → login : admin / mot de passe affiché par make build

# 4. Pour tout nettoyer et recommencer
make re

Pour tester si tout fonctionne : 
# Changer v1 → v2 dans ton repo GitHub, puis :
make sync       # force la synchronisation ArgoCD
curl http://localhost:8888/
# {"status":"ok", "message": "v2"}
